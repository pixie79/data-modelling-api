# Pre-commit Troubleshooting

## If pre-commit fails with sqlx database errors:

**Option 1: Generate sqlx metadata (recommended)**
```bash
# Start PostgreSQL (if using Docker)
docker-compose up -d postgres

# Set DATABASE_URL
export DATABASE_URL=postgresql://postgres:postgres@localhost:5432/data_modelling

# Generate metadata
./scripts/prepare-sqlx.sh

# Commit the .sqlx directory
git add .sqlx
git commit -m "Add sqlx offline metadata"
```

**Option 2: Skip pre-commit temporarily**
```bash
git commit --no-verify -m "Your commit message"
```

**Option 3: Set up a local PostgreSQL**
```bash
# Install PostgreSQL, then follow Option 1
```

Once `.sqlx` metadata files are committed, all developers can use pre-commit without a database.
